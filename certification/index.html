CERTIFICATION PROGRAM 2026

Secure Funder™

Technical Safeguards for Data Protection in Alternative Finance

This program verifies that your organization has appropriate technical safeguards to protect merchant data from unauthorized access and external sharing. Requirements focus on observable technical controls, not written policies. Where a specific control is not in place, functionally equivalent alternatives may be accepted.

Verification is conducted via non-recorded screenshare or on-site review at funder's expense. The process is collaborative — certification may proceed with agreed-upon remediation timelines or compensating controls rather than immediate disqualification.

1 Email Security

Multi-Factor Authentication (MFA)

Required: MFA must be enforced for all users with access to merchant data
Acceptable implementations: Google Workspace 2-Step Verification, Microsoft Entra ID MFA, or equivalent

Submission Handling

IF broker submissions are accepted via email, THEN one of the following is required:
Option A: Automated email processing with restricted human access
Submissions automatically forwarded/pushed to CRM without human intervention
Only management may access submission inbox for configuration/troubleshooting purposes
Processing personnel cannot access the submission inbox
If CRM is view-only (no downloads), watermarking not required
If CRM allows downloads, files must be watermarked before reaching CRM or at download
Option B: Email gateway to protect documents before human access
Dedicated inbox for broker submissions (e.g., submissions@company.com)
Attachments routed through gateway to automatically watermark files
Protected files sent to destination inbox for processing team (e.g., processing@company.com)
Secure Funder gateway available, or any provider acceptable

Outbound Email Controls

Required: Email rules that quarantine outbound emails with attachments for approval when sent to:
Personal email addresses (Gmail, Yahoo, Outlook.com, etc.)
External corporate domains
Note: This requirement applies to non-customer-facing roles (underwriters, processors, admin, etc.). Customer-facing roles (sales, account managers) may have legitimate needs to email externally and can be exempted from quarantine rules.
Required: Network-level blocking of personal email sites (mail.google.com, outlook.live.com, yahoo.com) on company devices
Required: Blocking of email forwarding rules to external addresses

2 Portal Security

Document Access Controls

IF files are available in your portal, THEN choose one:
Option A: View-only access to files (no download capability)
Option B: Files are watermarked before entering portal or at time of download

3 CRM Security

Multi-Factor Authentication

Required: MFA must be enabled for all users with access to merchant data

Document Access Controls

IF documents are stored in your CRM, THEN choose one:
Option A: View-only access to files (no download capability)
Option B: Files are watermarked before entering CRM or at time of download

Access Controls

Required: Personnel can only access accounts assigned to them
Role-based access controls must restrict visibility to assigned deals/accounts
Customer-facing sales roles must only be able to see their own deals
IF roles require access to entire database (e.g., managers, compliance, customer support), THEN:
Access to sensitive fields (SSN, DOB, EIN, contact information) restricted to roles requiring this information
Documents must be view-only OR watermarked (as addressed above)
IF full database download capability exists, THEN:
This function must be restricted to senior personnel only

4 Device & Network Management

Managed Devices

Required: All personnel accessing merchant data or documents must use company-managed devices (company-issued or managed personal devices with enforced security controls)

Device Security Controls

Required: USB ports restricted (storage devices blocked)
Required: Screen lock timeout (maximum 15 minutes of inactivity)
Required: Network restrictions prevent installing/uninstalling applications without IT approval
Recommended: Role-based printing controls (restrict printing to roles requiring hard copies)

Remote Access

Required: Remote employees must connect via company VPN when accessing email, CRM, or portals

Password Policy

Required: Documented password requirements including minimum complexity and periodic changes (quarterly recommended)

5 Third-Party Access

BPO / Offshore Teams

IF BPO or offshore teams have access to merchant documents (via email, CRM, portal, or cloud storage), THEN choose one:
Option A: View-only access to files (no download capability)
Option B: Documents are watermarked before third-party access

Syndication Programs

IF you operate a syndication program where participants access merchant information or documents, THEN:
Required: Provide written business justification for participant access to sensitive information
Required: Participants have view-only access OR documents are watermarked if downloadable

6 Cloud Storage

Cloud Service Usage

IF you use cloud storage services (Google Drive, Dropbox, OneDrive, Box, etc.) to house merchant documents, THEN:
Required: Disclose which services are used and who has access
Required: Users have view-only access OR documents are watermarked if downloadable

7 Personnel Controls

Background Checks

Required: All personnel with access to merchant documents must pass background checks appropriate for their role
Documentation of background check process will be reviewed during verification
Must demonstrate checks are conducted for all relevant personnel (redacted results acceptable)

Off-boarding Process

Required: Documented process for immediately revoking system access when employees leave or change roles
Process documentation will be reviewed during verification (sensitive information can be redacted)
