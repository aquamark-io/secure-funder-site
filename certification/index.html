Overview

Secure Funder™ verifies that your organization has appropriate technical safeguards to protect merchant data from unauthorized access and external sharing. Requirements focus on observable technical controls. Where a specific control is not in place, functionally equivalent alternatives may be accepted. Verification is conducted via (non-recorded) screenshare or on-site visit at funder's expense. The process is collaborative — certification may proceed with agreed-upon compensating controls rather than immediate disqualification.

1 Email Security

Multi-Factor Authentication

Required: MFA must be enabled

Submission Handling

IF broker submissions are accepted via email, THEN one of the following is required:
Option A: Automated email processing with restricted human access
Submissions automatically pushed to CRM without human intervention
Only management may access submission inbox for configuration/troubleshooting purposes
Processing personnel cannot access the submission inbox
Option B: Email gateway to protect documents before human access (Secure Funder gateway available at no cost)
Dedicated inbox for broker submissions (e.g., submissions@company.com)
Attachments routed through gateway to automatically watermark files
Protected files sent to destination inbox for processing team (e.g., processing@company.com)
Note: The Secure Funder gateway includes built-in watermarking and automatic delivery to your destination inbox—no API integration required. If using another provider, you'll need to integrate watermarking API and configure delivery of protected files.

Outbound Email Controls

Required: Email rules that quarantine outbound emails (with attachments) for approval when sent to:
Personal email addresses (Gmail, Yahoo, Outlook.com, etc.)
External corporate domains
Note: This requirement applies to back-office roles (underwriters, processors, etc.). Customer-facing roles (sales) and senior management may have legitimate needs to email externally and can be exempted from quarantine rules.
Required: Blocking of email forwarding rules to external addresses

2 Portal Security

Document Access Controls

Note: If you do not have a broker portal, skip this section.
IF files are available in your portal, THEN choose one:
Option A: View-only access to files (no download capability)
Option B: Files are watermarked before entering portal or at time of download

3 CRM Security

Multi-Factor Authentication

Required: MFA must be enabled

Document Access Controls

IF documents are stored in your CRM, THEN choose one:
Option A: View-only access to files (no download capability)
Option B: Files are watermarked before entering CRM or at time of download

Access Controls

Required: Personnel can only access accounts assigned to them
IF roles require access to entire database (e.g., managers, customer support), THEN:
Field-level access (SSN, DOB, EIN, contact information) restricted to roles requiring this information
Documents must be view-only OR watermarked (as addressed above)
IF full database download capability exists, THEN:
This function must be restricted to senior personnel only

4 Device & Network Management

Managed Devices

Required: Systems must be accessed from company-managed devices.

Device Security Controls

Required: USB ports: block removable storage devices
Required: Screen lock timeout (maximum 15 minutes of inactivity)
Required: Network restrictions prevent installing/uninstalling applications without IT approval
Required: Role-based printing controls (restrict printing to roles requiring hard copies)
Required: Network-level blocking of personal email sites (Gmail, Outlook, Yahoo, iCloud, etc)

Password Policy

Required: Copy of documented password requirements including minimum complexity and periodic changes (quarterly recommended)

5 BPO Access

On-Shore / Off-Shore Teams

Note: If you do not use BPO teams (on-shore or off-shore), skip this section.
IF BPO teams have access to merchant documents (via email, CRM, portal, or cloud storage), THEN choose one:
Option A: View-only access to files (no download capability)
Option B: Documents are watermarked before third-party access

6 Cloud Storage

Cloud Service Usage

IF you use cloud storage services (Google Drive, Dropbox, OneDrive, Box, etc.) to house merchant documents, THEN:
Required: Users have view-only access OR documents are watermarked if downloadable

7 Personnel Controls

Background Checks

Required: Personnel must pass background checks appropriate for their role
Provide background check authorization form for verification

Off-boarding Process

Required: Documented process for immediately revoking system access when employees leave or change roles
Provide a copy (sensitive or proprietary information can be redacted)
