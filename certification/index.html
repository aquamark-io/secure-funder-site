Overview

Secure Funder™ verifies that your organization maintains appropriate technical safeguards to protect sensitive information from unauthorized access and external sharing. Verification is conducted on-site or non-recorded screenshare, where controls are demonstrated in real time. The program does not require the use of any specific vendor or platform—organizations may use their existing tools, third-party solutions, or internally developed controls. Where a specific safeguard is not in place, functionally equivalent or compensating controls may be accepted.

1 Email Security

Multi-Factor Authentication

MFA must be enabled

Submission Handling

IF broker submissions are accepted via email, THEN one of the following is required:
Option A: Automated email processing with restricted human access
Submissions automatically pushed to CRM without human intervention
Only management may access submission inbox for configuration/troubleshooting purposes
Processing personnel cannot access the submission inbox
Option B: Use the Secure Funder Gateway to automatically protect submissions
No changes to broker submission inbox or workflow
Attachments routed through gateway to watermark files and embed metadata
Protected files sent to destination inbox for processing team (e.g., processing@yourcompany.com)

Outbound Email Oversight

Configure an email rule that copies messages to a monitoring mailbox when all of the following criteria are met:
Email is outbound
Message includes attachments
Recipients from external domain (outside your organization)
Sender is in a deal processing role (underwriters, processors)
Staff should be informed that outbound attachments are monitored. Known oversight encourages appropriate handling of sensitive information and helps deter unauthorized sharing. Emails are not blocked; copies are retained solely to support review or investigation if needed.
Note: Common implementations include Gmail Content Compliance ?

Gmail (Google Workspace) — external recipient + attachment (with monitoring copy)

  1. Admin console → Apps → Google Workspace → Gmail → Compliance → Content compliance → Add rule.
  2. Set Email messages to affect: ☑ Outbound.
  3. Under Expressions (set to ALL must match), click Add → choose Advanced content match and add these expressions:
    • A) Attachment present
      • Location: Raw message
      • Match: Contains text
      • Value: Content-Disposition: attachment
    • B) External recipient exists (at least one recipient)
      • Location: Any envelope recipient
      • Match: Matches regex
      • Value: @
    • C) Exclude internal-only messages
      • Location: Any envelope recipient
      • Match: Not matches regex
      • Value: @yourcompany\.com$
  4. Under actions → Also deliver to → add your monitoring mailbox (e.g., monitoring@yourcompany.com).
    • ☑ Do not deliver spam to this recipient
    • ☑ Suppress bounces from this recipient
  5. Scope the rule to the appropriate Group or OU (recommended: group for back-office).
  6. Save.

Microsoft 365 (Exchange Online) — Monitor outbound attachments to external recipients

This setup copies outbound emails with attachments sent to any external recipient to a monitoring mailbox.

Step-by-step
  1. Go to Exchange admin center → Mail flow → Rules.
  2. Click + Add rule → Create a new rule.
  3. Name the rule (e.g., Outbound attachments to external recipients).
  4. Apply this rule if…
    • The sender is located → Inside the organization
    • The recipient is located → Outside the organization
    • Any attachment → Yes
  5. Do the following…
    • Add a blind carbon copy (Bcc) to → your monitoring mailbox (e.g., monitoring@yourcompany.com)
  6. (Optional but recommended)
    • Add a policy tip or disclaimer indicating that external attachments may be monitored.
  7. Except if… (optional)
    • Exclude specific users or groups if needed (e.g., executives).
  8. Save the rule.

2 Portal Security

Access Controls

Note: If you do not have a portal, skip this section.
IF sensitive information is accessible in your portal (e.g. SSN), THEN:
Controls to deter capture of on-screen data (e.g., screenshots disabled or screen watermarking enabled)
IF files are available in your portal, THEN choose one:
Option A: View-only access to files (no download capability)
Option B: Files are watermarked before entering portal or at time of download

3 CRM Security

Multi-Factor Authentication

MFA must be enabled

Access Controls

System access:
Users may access CRM only from company-managed devices. Access from personal devices is not permitted.
Role-based access:
Personnel can only access accounts and information required for their role
Sensitive fields (e.g., SSN) are accessible only to designated roles, with controls in place to deter capture of on-screen data (e.g., screenshots disabled or screen watermarking enabled).
User verification:
Users must be from your domain, with exception of service-level access (developers, vendors)
External domain access must be justified when associated with other funders, brokers, or software platforms involved in alternative finance deal processing.
IF documents are stored in your CRM, THEN choose one:
Option A: View-only access to files (no download capability)
Option B: Files are watermarked before entering CRM or at time of download
IF full database download capability exists, THEN:
This function must be restricted to senior personnel only

4 Device & Network Management

Managed Devices

Systems must be accessed from company-managed devices.

Device Security Controls

Block removable storage devices (USB)
Restrict access to Task Manager
Screen lock timeout (maximum 15 minutes of inactivity)
Restrictions to prevent installing/uninstalling applications without IT approval
Restrict printing to only roles requiring hard copies
Network-level blocking of personal email sites (Gmail, Outlook, Yahoo, iCloud, etc)

Password Policy

Documented requirements including minimum complexity and periodic changes (quarterly recommended)

5 BPO Access

On-Shore / Off-Shore Teams

Note: If you do not use BPO teams (on-shore or off-shore), skip this section.
IF BPO teams have access to merchant documents (via email, CRM, portal, or cloud storage), THEN choose one:
Option A: View-only access to files (no download capability)
Option B: Documents are watermarked before third-party access

6 Cloud Storage

Cloud Service Usage

IF you use cloud storage services (Google Drive, Dropbox, OneDrive, Box, etc.) to house merchant documents, THEN:
Users have view-only access OR documents are watermarked if downloadable

7 Personnel Controls

Background Checks

Personnel must pass background checks appropriate for their role
Provide background check authorization form for verification

Workstation Policy

Personal devices may not be used while conducting business activities at designated workstations. This includes cell phones, tablets, and laptops. This restriction applies only to designated workstations; personal device use is permitted in break rooms, common areas, and other non-processing areas.

Off-boarding Process

Documented process for immediately revoking system access when employees leave or change roles
