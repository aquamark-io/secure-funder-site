Email Security

Multi-Factor Authentication

MFA must be enabled

Submission Handling

IF broker submissions are accepted via email, THEN one of the following is required:
Option A: Automated email processing with restricted human access
Submissions automatically pushed to CRM without human intervention
Only management may access submission inbox for configuration/troubleshooting purposes
Processing personnel cannot access the submission inbox
Option B: Use the Secure Funder Gateway if email submissions require human handling
No changes to broker submission inbox or workflow
Broker submits, then attachments are routed through gateway to watermark files and embed metadata
Files are then sent back to inbox of your choice for human processing
Learn about the gateway
Option C: Submissions accepted only via portal (no email submissions)
Broker sends documents through a portal that restricts downloads and access
2

Access Controls

Role-Based Access

Access must be restricted based on job function

Administrative Controls

Admin access must be limited to management

Least Privilege

Users may only access systems and data necessary for their role
4

Document Controls

Download Restrictions

IF staff can download documents from any system (CRM, inbox, portals), THEN one of the following is required:
Option A: View-only access for processing staff
Processing staff can view documents but cannot download/export
Option B: Watermark documents used internally
Documents used for processing must include visible attribution and embedded metadata

Storage & Sharing

Documents may not be stored or shared outside approved systems
5

Personnel Controls

Background Checks

Personnel must pass background checks appropriate for their role

Password Policy

Documented requirements including minimum complexity and periodic changes

Screen Security

Screen lock required when away from workstation

Workstation Policy

Personal cell phones may not be used at workstations where customer data is visible

Off-boarding Process

Documented process for immediately revoking system access when employees leave or change roles
