Email security

MFA + how submissions are handled

Multi-factor authentication

MFA is enabled on email accounts used for submissions and operations.

Submission handling

If broker submissions are accepted via email, one of the following paths must be used:

Option A — Automated email processing with restricted human access
Submissions are automatically pushed to your CRM without staff handling the inbox.
  • Only management may access the submission inbox for configuration/troubleshooting
  • Processing personnel cannot access the submission inbox
Option B — Secure Funder Gateway (for inbox workflows that require human handling)
Broker submits as usual. Attachments are routed through the gateway to watermark + embed metadata, then delivered to a processing inbox of your choice.
Option C — Portal-only submissions (no email submissions)
Brokers submit documents through a portal workflow that restricts access and handling.
2

CRM / Portal

Access controls + role restrictions inside your systems

Access controls

Role-based access is enforced (access restricted based on job function).

Admin controls are limited to management.

Least privilege (users only access systems/data necessary for their role).

Submission path (portal)

If you use a portal, access is controlled and aligned to staff roles (no broad sharing).

4

Document controls

Downloads, internal handling, and where files can live

Download restrictions

If staff can download documents from any system (CRM, inbox, portals), one of the following is required:

Option A — View-only access for processing staff
Processing staff can view documents but cannot download/export.
Option B — Watermark documents used internally
Documents used for processing include visible attribution + embedded metadata.

Storage & sharing

Documents aren’t stored or shared outside approved systems.

5

Personnel controls

Baseline policies for users and workstations

Baseline requirements

Background checks appropriate for role.

Password policy (documented requirements including minimum complexity and periodic changes).

Screen lock when away from workstation.

No personal cell phones used at workstations where customer data is visible.

Off-boarding process to immediately revoke system access when employees leave or change roles.
